# Group / Organization Attribution — Global Terrorism Database

Machine learning pipeline that predicts the likely **perpetrating group/organization**
from historical event characteristics (attack type, weapon, target, region, casualties,
etc.), trained on the [Global Terrorism Database](https://www.start.umd.edu/gtd/)
(170,000+ real-world incident records, 1970–2016).

This is an **event-pattern classification** project — it predicts which of a fixed set
of well-documented historical groups an event's characteristics most resemble. It does
not identify, profile, or track any individual person, and it cannot attribute a real,
unsolved attack.

## Results

| Metric | Score |
|---|---|
| Test Accuracy | 92.6% |
| Test Macro-F1 | 0.901 |
| 3-fold CV Macro-F1 (mean ± std) | 0.895 ± 0.002 |
| Best hyperparameters | `n_estimators=100, max_depth=15, min_samples_leaf=3` |
| Groups covered | 30 |

Model: Random Forest, tuned via `GridSearchCV`, validated via `StratifiedKFold`
cross-validation, evaluated on a held-out test set.

## How it was built

1. **Data cleaning** — dropped incomplete records and handled missing fatality/injury
   counts. Also dropped incidents with an **"Unknown"** perpetrator (~45% of all rows),
   since that's not a real organization — it's the absence of an attribution, and
   leaving it in would let the model "win" by mostly predicting the largest bucket.
2. **Class filtering** — kept only groups with **500+** historical incidents (30 groups
   survive this cut, out of several hundred candidates in the raw data). Below that
   threshold there just isn't enough history for the model to learn a real pattern —
   many groups in the raw data have only 1-2 recorded incidents total.
3. **Feature engineering** — label-encoded region, country, target type, weapon type,
   and **attack type** (previously the target, now an input feature since group is the
   new target); combined with year, month, casualty counts, and success flag.
4. **Train/test split** — stratified 80/20 split to preserve class balance.
5. **Hyperparameter tuning** — `GridSearchCV` over `n_estimators`, `max_depth`, and
   `min_samples_leaf` with 3-fold stratified CV, optimizing macro-F1. `min_samples_leaf`
   matters a lot here: with 30 target classes, each tree node stores a per-class count
   array, so unconstrained trees get very large on disk without it.
6. **Cross-validation** — final model re-validated with `cross_val_score` (3-fold)
   to confirm stability (std ≈ 0.002 — consistent across folds).
7. **Evaluation** — classification report + accuracy/macro-F1 on a held-out test set.
8. **Deployment** — trained model + encoders saved with `joblib` (~78 MB), served
   through a Streamlit app for interactive predictions.

## Run it locally

```bash
pip install -r requirements.txt

# Train the model (runs in 3 checkpointed stages)
python src/train_model.py prep
python src/train_model.py tune
python src/train_model.py finalize

# Launch the app
streamlit run src/app.py
```

## Limitations

- Data covers 1970–2016; the official GTD has since been updated but requires
  registration for newer years.
- The model only covers the **30 groups** with enough historical incidents (500+) to
  learn a reliable pattern from — it will never predict a group outside that list, and
  it cannot say anything about a group's very first attacks before it built up history.
- `region_txt` / `country_txt` are strong predictors here because many groups operate
  in a specific geographic area (e.g. FARC ≈ Colombia, Taliban ≈ Afghanistan) — worth
  noting if asked, similar in spirit to the earlier `Weapons` ↔ `Attack Type` overlap.
- The very high scores on some classes (e.g. Taliban, FARC) partly reflect how
  geographically concentrated those groups' historical activity is, not that the model
  has learned some deeper tactical "signature" independent of location.
- This is a portfolio/educational project, not a production threat-attribution system.
