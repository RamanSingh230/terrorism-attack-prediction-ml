# Attack Pattern Prediction — Global Terrorism Database

Machine learning pipeline that predicts the **type of attack** (bombing, armed assault,
kidnapping, etc.) from historical event characteristics, trained on the [Global Terrorism
Database](https://www.start.umd.edu/gtd/) (170,000+ real-world incident records, 1970–2016).

This is an **event-pattern classification** project — it predicts aggregate attack
characteristics from historical data. It does not identify, profile, or track any
individual person.

## Results

| Metric | Score |
|---|---|
| Test Accuracy | 83.9% |
| Test Macro-F1 | 0.654 |
| 3-fold CV Macro-F1 (mean ± std) | 0.655 ± 0.005 |
| Best hyperparameters | `n_estimators=200, max_depth=15` |

Model: Random Forest, tuned via `GridSearchCV`, validated via `StratifiedKFold`
cross-validation, evaluated on a held-out test set.

## How it was built

1. **Data cleaning** — dropped incomplete records, handled missing fatality/injury
   counts, filtered rare attack-type classes for reliable cross-validation.
2. **Feature engineering** — label-encoded region, country, target type, weapon type,
   and group; combined with year, month, casualty counts, and success flag.
3. **Train/test split** — stratified 80/20 split to preserve class balance.
4. **Hyperparameter tuning** — `GridSearchCV` over `n_estimators` and `max_depth`
   with 3-fold stratified CV, optimizing macro-F1.
5. **Cross-validation** — final model re-validated with `cross_val_score` (3-fold)
   to confirm stability (std ≈ 0.005 — consistent across folds).
6. **Evaluation** — classification report + accuracy/macro-F1 on a held-out test set.
7. **Deployment** — trained model + encoders saved with `joblib`, served through a
   Streamlit app for interactive predictions.

## Run it locally

```bash
pip install -r requirements.txt

# Train the model (runs in 3 checkpointed stages)
python src/train_model.py prep
python src/train_model.py tune
python src/train_model.py finalize

# Launch the app
streamlit run src/app.py
```

## Limitations

- Data covers 1970–2016; the official GTD has since been updated but requires
  registration for newer years.
- `Weapons` is the strongest predictor and correlates closely with `Attack Type`
  by definition — a known limitation worth noting if asked about feature leakage.
- This is a portfolio/educational project, not a production threat-detection system.
